# Slate — менеджер скриптов для After Effects и Premiere Pro

Десктоп-приложение (Electron) под Windows и macOS. Слева — вертикальный рейл с
найденными хостами (Ae, Pr); внизу рейла — Аккаунт и Настройки. Внутри вкладки
программы показан каталог всех скриптов и паков для неё: купленные выделены и
устанавливаются в один клик, некупленные показаны с ценой. Поиск — только внутри
вкладки программы.

## Запуск

Нужен Node.js LTS (с ним идёт npm).

```bash
npm install
npm start
```

Открыть на Windows и на macOS можно одинаково. Если Adobe не установлен (или вы
просто смотрите интерфейс), приложение покажет демо-хосты, а установка пойдёт в
безопасную папку внутри userData — так UI всегда наполнен и кликабелен.

## Сборка инсталляторов (позже)

```bash
npm run dist:mac   # .dmg
npm run dist:win   # .exe (nsis)
```

Подпись кода (EV-сертификат на Windows, нотаризация Apple) — отдельный шаг, тесно
связанный с защитой; сейчас не настроена.

## Структура

```
src/
  main/        — главный процесс (Node): окно, IPC, сеть, файловые операции
    index.js          окно + IPC + загрузка манифеста + установка
    config.js         источник релизов: local или remote (+ URL манифеста)
    detect.js         поиск установленных AE/Premiere + папка CEP/extensions
    catalog.js        каталог (owned/локально; позже — entitlements бэкенда)
    releases.js       манифест релизов: local (из файлов) или remote (по HTTP)
    download.js        httpGetJson + загрузка с SHA-256 и распаковкой
    storage-server.js  локальное «хранилище» для режима local (стенд-ин S3/CDN)
    store.js          локальное состояние «что установлено»
  preload/
    index.js     безопасный мост contextBridge (белый список IPC)
  renderer/      — интерфейс (без доступа к Node)
    index.html   оболочка + строгий CSP + спрайт иконок
    styles.css   тёмная тема как в AE/Premiere, янтарный акцент
    renderer.js  рейл, каталог по хосту, загрузка, проверка обновлений
tools/make-manifest.js — считает sha256/размер артефакта для manifest.json
manifest.example.json  — шаблон манифеста для загрузки на GitHub
resources/scripts/     — артефакт AEMotionPanel.zip (раздаётся в режиме local)
```

## Загрузка плагинов из хранилища

Установка идёт настоящим сетевым пайплайном (всё в main-процессе):

1. собирается манифест релиза: URL файла + ожидаемый `sha256` + версия;
2. файл скачивается стримом во временный файл, попутно считается SHA-256;
3. контрольная сумма сверяется с ожидаемой (проверка целостности);
4. при совпадении файл **атомарно** переносится в папку хоста; при несовпадении —
   установка отменяется, в папку хоста ничего не пишется.

Источник манифеста выбирается (`config.js`): **local** — локальный сервер на
`127.0.0.1` (`storage-server.js`) раздаёт файлы из `resources/scripts` (офлайн);
**remote** — `manifest.json` берётся по HTTP с GitHub (см. раздел «Хранилище
плагинов»). Код загрузчика (`download.js`) для обоих одинаков — он уже умеет
https и редиректы CDN. Дальше тем же кодом источником станет твой бэкенд
(`GET /releases/latest`), который после проверки лицензии вернёт короткоживущий
**подписанный URL** — клиент не меняется.

Как сделать «по-взрослому» (что уже есть и что добавить):

- качать в main, не в renderer — **есть**;
- проверка SHA-256 + атомарная установка — **есть**;
- https + следование редиректам presigned/CDN — **есть**;
- подписанный короткоживущий URL вместо постоянной публичной ссылки — на бэкенде;
- подпись файла (Ed25519/RSA) + проверка публичным ключом в клиенте — сильнее
  хеша, защищает даже от скомпрометированного хранилища (добавить);
- многофайловые/CEP-плагины как `.zip` с распаковкой — **есть** (через `adm-zip`,
  он в зависимостях);
- кэш скачанных версий и докачка для больших файлов — добавить.

## Хранилище плагинов (GitHub) и источник релизов

Плагины лежат отдельно от приложения. Источник релизов переключается в
`src/main/config.js` (или через переменные окружения):

- `RELEASE_SOURCE = 'local'` — артефакты из встроенной папки `resources/scripts`,
  раздаются локальным сервером. Работает из коробки и офлайн.
- `RELEASE_SOURCE = 'remote'` — приложение берёт `manifest.json` по
  `REMOTE_MANIFEST_URL` (GitHub raw / Pages / любой HTTPS), а артефакт качает по
  ссылке из манифеста и проверяет SHA-256.

**Почему GitHub, а не Яндекс.Диск:** у GitHub стабильные прямые ссылки (raw и
Release-ассеты) и версионность из коробки; у Яндекс.Диска прямую ссылку надо
получать отдельным вызовом API (публичная ссылка → временный href). Прод-цель
дальше — S3 / Cloudflare R2 с подписанными ссылками (это и есть механизм
лицензирования). Для запуска сейчас GitHub оптимален.

### Настройка за 5 шагов

1. Создай репозиторий, например `slate-releases` (пока публичный).
2. Положи `AEMotionPanel.zip` в папку `plugins/` репозитория (для крупных файлов
   позже используй Release-ассеты — лимит raw-файла 100 МБ, ассета 2 ГБ).
3. Сгенерируй запись манифеста:
   `node tools/make-manifest.js resources/scripts/AEMotionPanel.zip https://raw.githubusercontent.com/<owner>/<repo>/main/plugins/AEMotionPanel.zip motionpanel`
   — впиши `version`, собери из этого `manifest.json` (шаблон — `manifest.example.json`)
   и положи его в корень репозитория.
4. Возьми raw-ссылку манифеста:
   `https://raw.githubusercontent.com/<owner>/<repo>/main/manifest.json`
5. Включи remote: в `config.js` поставь `RELEASE_SOURCE: 'remote'` и
   `REMOTE_MANIFEST_URL` = ссылка из п.4 (или запусти
   `SLATE_RELEASE_SOURCE=remote SLATE_MANIFEST_URL=… npm start`).

Готово — приложение скачивает плагин из интернета.

### Шов для защиты

`manifest.json` сейчас публичный (версии + публичные ссылки), а владение (`owned`)
— локальный флаг. В проде этот же манифест начнёт отдавать **твой бэкенд**
(`GET /releases/latest`): он проверит лицензию и окно поддержки и вернёт
короткоживущий **подписанный URL** для нужной версии. Клиентский код (загрузка →
проверка хеша → установка) не меняется — меняется только источник манифеста. Для
приватного хранилища токен в клиент не зашивается: за ним пойдёт бэкенд-прокси.

## Установка CEP-расширения и режим разработчика

Motion Panel — это CEP-расширение (`com.motionpanel.base`, хост After Effects).
Ставится распаковкой папки `AEMotionPanel` в общую папку CEP-расширений:

- **Windows:** `%APPDATA%\Adobe\CEP\extensions\`
- **macOS:** `~/Library/Application Support/Adobe/CEP/extensions/`

Так как расширение пока не подписано (ZXP), AE загрузит его только в **режиме
разработчика CEP**. В приложении для этого есть кнопка «Включить режим
разработчика» (на вкладке After Effects) — она выставляет `PlayerDebugMode = 1`
для CSXS 9–12. Вручную это:

- **Windows:** в реестре `HKCU\Software\Adobe\CSXS.<N>` строковый `PlayerDebugMode = 1`
- **macOS:** `defaults write com.adobe.CSXS.<N> PlayerDebugMode 1`

После установки: перезапустить AE → Window → Extensions → Motion Panel.

## Проверка обновлений

Версия плагина — это `ExtensionBundleVersion` из `CSXS/manifest.xml`. Приложение
читает **установленную** версию прямо с диска (из манифеста в `CEP/extensions`), а
**доступную** берёт из релиз-манифеста (в проде — из ответа бэкенда
`/releases/latest`) и сравнивает семантически (`compareSemver`).

Проверка версий работает так:

- **автоматически** — при запуске, периодически (раз в 20 минут) и при возврате
  фокуса в окно (не чаще раза в 5 минут);
- **бейдж** — на иконке хоста в рейле появляется точка, если для него есть
  обновление, так что заметно без захода во вкладку;
- **что нового** — на карточке обновления видно переход версий (`v0.1.0 → v0.2.0`)
  и заметки релиза из поля `notes` манифеста;
- **вручную** — кнопка «Проверить обновления» перечитывает манифест по сети;
- **версия приложения** — в Настройках берётся из необязательного блока `app`
  манифеста; если она новее текущей, показывается «Доступна версия …» со ссылкой.

Чтобы выпустить новую версию плагина: подними `ExtensionBundleVersion` в манифесте
плагина, собери новый `AEMotionPanel.zip`, положи в хранилище и обнови
`version`/`sha256`/`notes` в релиз-манифесте. Клиент покажет обновление сам.
Проверить сейчас: в `manifest.json` (или `catalog.js` для local) поставь версию
выше установленной — появятся бейдж и карточка «Обновить».

## Где встаёт защита (этап 2)

Места уже размечены в коде комментариями «ЗАЩИТА»/«место под защиту»:

- **Auth-гейт** — экран входа перед приложением; сейчас приложение открывается
  сразу. Кнопки «Выйти»/«Деактивировать» — заглушки.
- **Проверка лицензии перед установкой** — в `ipcMain.handle('script:install')`
  стоит заглушка-комментарий; сюда сядет проверка владения и окна поддержки.
- **Окно поддержки (гейт на обновления)** — поле `supportUntil` в каталоге и
  состояние «Поддержка истекла → Продлить» (ProxyPro). На бэкенде это решает,
  отдавать ли новую версию.
- **Привязка устройства** — блок «Устройства» в Аккаунте (1 из 1, отпечаток,
  деактивация). Здесь будет node-lock и активация/деактивация слота.
- **Каталог из entitlements** — `catalog.js` сейчас мок; флаг `owned` и список
  скриптов придут с сервера после логина.

## Что реально работает сейчас

- детект AE/Premiere на Win/Mac (с демо-фолбэком);
- навигация по хостам, аккаунту, настройкам; тёмная/светлая тема;
- каталог по хосту с купленными/некупленными; поиск внутри вкладки;
- установка: реальная загрузка артефакта из хранилища по сети с прогрессом,
  проверкой SHA-256 и атомарной установкой. Для Motion Panel (CEP) — скачивается
  `AEMotionPanel.zip`, проверяется хеш и распаковывается в общую папку
  `CEP/extensions`; установленная версия читается с диска из `CSXS/manifest.xml`;
- состояние «установлено» переживает перезапуск.

## Известные нюансы

- На Windows папка `Program Files\...` требует прав администратора — установка
  туда может вернуть ошибку прав. Решения (per-user установка / запрос
  elevation) — на этапе доводки.
- «Купить» открывает заглушечный URL `example.com/buy/...` — заменим на реальную
  страницу оплаты.
- Версия Electron в `package.json` указана диапазоном — при желании обновите.
